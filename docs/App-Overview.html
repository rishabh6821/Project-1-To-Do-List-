To Do List App — Overview

This document explains the app structure, logic, storage, backend behavior, and how to run and test the project.

Project layout (important files)

  • index.html / dist/index.html — App entry HTML.
  • src/main.jsx — React mount point.
  • src/Options.jsx — Main logic: users, tasks, storage, admin view.
  • src/App.jsx — To Do UI for incomplete tasks.
  • src/Completed.jsx — Completed tasks UI.
  • src/AllWorks.jsx — All tasks listing.
  • api/tasks.js — Serverless API handler (demo).

High-level behavior

Authentication & users

  • Users stored in localStorage under todo.users.
  • Admin user auto-created if missing (Admin / admin123).
  • Logged-in user saved as todo.currentUser for session restore.
  • Password changes update todo.users client-side.

Tasks

  • Per-user tasks stored at todo.tasks.<username> in localStorage.
  • Import/export available (File System Access API or JSON download/upload).

Task operations — addTask(), removeTask(), toggleComplete() update React state and persist to localStorage.

Backend (api/tasks.js)

This demo serverless handler exposes REST endpoints for tasks. It uses an in-memory store and optionally writes to data/tasks.json locally. Note: serverless file storage is ephemeral.

Security notes

  • Passwords currently stored in plaintext in localStorage — not secure.
  • Consider moving authentication and tasks to a secure backend with hashed passwords and a proper DB.
